## Question 3
**Q:** Write a short technical article (3 paragraphs) in English about the differences between SQL and NoSQL databases. The article should be suitable for a developer audience and cover: data structure, scalability, and use cases.

**A:**
**SQL vs. NoSQL Databases: Key Differences for Developers**  

SQL (Structured Query Language) databases, such as MySQL, PostgreSQL, and Oracle, rely on a rigid, tabular structure with predefined schemas. Data is organized into rows and columns, enforced through data types and relationships via foreign keys. This structured approach ensures consistency and supports complex queries using SQL, making them ideal for applications requiring transactional integrity, like financial systems. In contrast, NoSQL databases (e.g., MongoDB, Cassandra, Redis) prioritize flexibility with unstructured or semi-structured data models, including documents, key-value pairs, or graphs. Their schema-less design allows dynamic schema changes, simplifying development for evolving data requirements but requiring developers to manage data normalization manually.  

Scalability is another critical distinction. SQL databases traditionally scale vertically (adding more resources to a single server), which can become a bottleneck under high load. NoSQL databases, however, are designed for horizontal scaling, distributing data across multiple nodes or clusters. This makes them well-suited for distributed systems and big data applications, though at the cost of eventual consistency (as per CAP theorem) compared to SQL’s ACID compliance. Developers must weigh trade-offs between scalability and consistency based on their use case.  

Use cases further highlight these differences. SQL databases are preferred for applications demanding complex joins, transactions, and strong consistency, such as ERP systems or banking platforms. NoSQL excels in scenarios requiring high write throughput, flexible schemas, or real-time analytics, like IoT data ingestion, social media platforms, or content management systems. For example, a real-time recommendation engine might use a document-store NoSQL database for fast, schema-free data access, while a stock trading platform would rely on SQL to enforce transactional accuracy. Understanding these trade-offs helps developers choose the right tool for their application’s needs.

**Q:** Bir Python projesinde Hermes mesaj kütüphanesini kullanarak publisher/subscriber pattern'i uyguluyorsun.

Aşağıdakileri açıkla ve kod örneği ver:
1. Publisher nasıl mesaj yayınlar?
2. Subscriber nasıl dinler?
3. Bu pattern'in doğrudan fonksiyon çağrısına göre avantajı nedir?

Cevabı Türkçe ver.

**A:**
### 1. **Publisher Nasıl Mesaj Yayınlar?**  
Hermes kütüphanesi kullanılarak bir **Publisher**, belirli bir konuya (topic) mesaj gönderir. Bu işlem genellikle bir `publish` metodunu çağırarak yapılır. Aşağıda basit bir örnek verilmiştir:

```python
from hermes import Hermes

# Hermes örneği oluştur
hermes = Hermes(broker_url="redis://localhost:6379/0")  # Redis gibi bir mesaj broker'ı kullanılır

# Mesajı yayınla
hermes.publish(topic="haberler", message={"baslik": "Yeni özellik eklendi!", "detay": "Hermes 2.0 versiyonu çıktı."})
```

**Açıklama:**  
- `broker_url`: Mesajların iletilceği broker (Redis, RabbitMQ vb.) belirtilir.  
- `topic`: Dinleyici (subscriber) bu konuya abone olur.  
- `message`: Gönderilecek veri (genellikle JSON formatında olur).

---

### 2. **Subscriber Nasıl Dinler?**  
**Subscriber**, belirli bir konuya abone olur ve gelen mesajları işler. Hermes'te bu, bir fonksiyonu `@hermes.subscribe` dekoratörü ile etiketleyerek yapılır:

```python
from hermes import Hermes

hermes = Hermes(broker_url="redis://localhost:6379/0")

# "haberler" konusunu dinleyen fonksiyon
@hermes.subscribe(topic="haberler")
def handle_news(message):
    print("Yeni haber alındı:")
    print(f"Başlık: {message['baslik']}")
    print(f"Detay: {message['detay']}")

# Dinleme başlat (asenkron olarak çalışır)
hermes.start_listening()
```

**Açıklama:**  
- `@hermes.subscribe(topic="haberler")`: `handle_news` fonksiyonu, "haberler" konusundaki mesajları işler.  
- `hermes.start_listening()`: Dinleme süreci başlatılır. Bu, genellikle asenkron (örneğin `asyncio` ile) çalışır.

---
