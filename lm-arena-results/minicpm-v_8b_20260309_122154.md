## Question 3
**Q:** Write a short technical article (3 paragraphs) in English about the differences between SQL and NoSQL databases. The article should be suitable for a developer audience and cover: data structure, scalability, and use cases.

**A:**
SQL and NoSQL databases are two fundamentally different types of databases, with distinct approaches to data structure, scalability, and use cases. In this article, we'll explore these differences and explain how they impact developers when selecting a database solution.

Data Structure:

SQL databases, such as relational databases, use a structured approach to data organization. Data is stored in tables, which contain rows and columns, with each row representing an individual record and each column representing a specific field. This structured approach allows for strong data integrity, consistency, and ease of querying, making it ideal for scenarios where data relationships and complex queries are necessary.

NoSQL databases, on the other hand, embrace a more flexible, schema-less, and dynamic data structure. Data is often stored in key-value pairs or JSON documents, which allows for greater flexibility in how data is stored and accessed. NoSQL databases excel in handling unstructured, semi-structured, and complex data, making them suitable for scenarios where data is frequently changing or where data relationships are not well-defined.

Scalability:

SQL databases are generally designed for ACID compliance (atomicity, consistency, isolation, and durability), which ensures reliable and consistent data management. However, this structure often results in limitations for scaling out horizontally, meaning that adding more hardware to handle increased load can be challenging and expensive. SQL databases often rely on centralized data stores, which can become a bottleneck as the amount of data and the number of users increase.

NoSQL databases, in contrast, are designed with scalability in mind. They often support horizontal scaling, meaning that it's easier to add more nodes to handle increased load. This scalability is achieved through distributed architectures, which distribute data across multiple servers, making it possible to handle large volumes of data and users. NoSQL databases typically have simpler and more flexible data models, which also contribute to their scalability.

Use Cases:

SQL databases are well-suited for scenarios where data relationships are well-defined, and complex queries are required. Examples include transactional systems, such as e-commerce platforms, where data consistency and integrity are crucial. SQL databases are also commonly used in traditional applications like customer relationship management (CRM) systems, enterprise resource planning (ERP) systems, and financial systems.

NoSQL databases, on the other hand, are ideal for scenarios where data is frequently changing, or where data relationships are not well-defined. Examples include big data analytics, real-time data processing, and web applications. NoSQL databases excel in handling large volumes of unstructured data, such as social media data, sensor data, and clickstream data.

## Question 5
**Q:** Configure iptables on an Ubuntu server according to these rules and explain each line:

- Drop all incoming traffic by default
- Accept SSH (port 22) only from 10.0.0.0/24
- Accept HTTP (80) and HTTPS (443) from anywhere
- Allow all outgoing traffic
- Make the rules persistent

Please answer in English.

**A:**
To configure iptables on an Ubuntu server according to the given rules, we'll follow these steps:
1. Drop all incoming traffic by default:
```bash
iptables -A INPUT -j DROP
```
This command drops all packets in the INPUT chain, effectively blocking all incoming traffic.
1. Accept SSH (port 22) only from 10.0.0.0/24:
```bash
iptables -A INPUT -p tcp --dport 22 -s 10.0.0.0/24 -j ACCEPT
```
This command allows TCP connections to port 22 only from the specified IP range (10.0.0.0/24).
1. Accept HTTP (80) and HTTPS (443) from anywhere:
```bash
iptables -A INPUT -p tcp --dport 80,443 -j ACCEPT
```
This command allows TCP connections to ports 80 and 443 from any source IP address.
1. Allow all outgoing traffic:
```bash
iptables -A OUTPUT -j ACCEPT
```
This command allows all packets in the OUTPUT chain, effectively allowing all outgoing traffic.
1. Make the rules persistent:
```bash
service iptables save
```
This command saves the current state of iptables rules to the `/etc/iptables/rules.v4` file, ensuring that the rules persist even after the server restarts.
With these commands, the iptables rules on the Ubuntu server will be configured according to the given rules. The `iptables -L` command can be used to verify the current rules.
