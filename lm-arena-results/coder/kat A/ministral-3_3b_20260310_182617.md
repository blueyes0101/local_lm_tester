**A:**
Here’s a structured breakdown of each requirement, including implementation details for a **Redis-based refresh token system** with token reuse detection and password change invalidation:

---

### **1. `/auth/refresh` (POST)**
**Purpose**: Exchange a valid refresh token (from cookie) for a new access token.
**Steps**:
1. Extract refresh token from cookie (`refresh_token`).
2. Validate:
   - Token exists in Redis.
   - Token is not expired (`TTL` check).
   - Token is not revoked (blacklisted or invalidated).
3. Rotate:
   - Invalidate old refresh token (set `TTL=0` or delete).
   - Issue new refresh token (store in Redis with `TTL=7d`).
   - Set new access token (short-lived, e.g., `TTL=15m`) in response.
4. **Response**:
   ```json
   {
     "access_token": "new_jwt",
     "refresh_token": "new_refresh_token",
     "expires_in": 900
   }
   ```

---

### **2. `/auth/logout` (POST)**
**Purpose**: Invalidate refresh token and clear cookies.
**Steps**:
1. Extract refresh token from cookie.
2. Invalidate it in Redis:
   - Set `TTL=0` (or delete) to mark as revoked.
3. Clear cookies:
   - `Set-Cookie: refresh_token=; Max-Age=0; Path=/`.
4. **Optional**: Logout from other devices (e.g., via blacklist).

---

### **3. Redis-Based Token Store**
**Data Structure**:
- **Key**: `refresh_token:<uuid>`
  - **Value**: JSON object with:
    ```json
    {
      "user_id": "123",
      "created_at": "2026-03-10T12:00:00Z",
      "expires_at": "2026-03-17T12:00:00Z",
      "revoked": false,
      "ip_address": "192.168.1.1"  // Optional: track device
    }
    ```
- **TTL Strategy**:
  - **Refresh Tokens**: `7d` (long-lived, but short enough to rotate).
  - **Access Tokens**: `15m` (short-lived, stateless).
  - **Blacklisted Tokens**: `TTL=0` (immediate revocation).

**Example Redis Commands**:
```bash
# Store new refresh token
SET refresh_token:abc123 "{\"user_id\":\"123\",\"created_at\":\"2026-03-10T12:00:00Z\",\"expires_at\":\"2026-03-17T12:00:00Z\"}" EX 604800

# Invalidate old refresh token
SET refresh_token:abc123 "{\"revoked\":true}" EX 0
```

---

### **4. Handling Token Reuse Detection**
**Problem**: A user submits the same refresh token twice (e.g., due to network issues).
**Solution**: Track usage per token in Redis:
- **Key**: `refresh_token:<uuid>_usage`
- **Value**: Increment counter (e.g., `1` for first use, `2` for second).
- **Action**:
  - If `usage > 1`, reject the request (e.g., return `403 Forbidden`).
  - **Alternative**: Log the event for debugging.

**Example**:
```bash
# After first use
INCR refresh_token:abc123_usage
# After second use (reject)
GET refresh_token:abc123_usage  # Returns 2 → reject
```

---

### **5. Password Change Invalidation**
**Purpose**: Invalidate all sessions for a user when their password changes.
**Steps**:
1. **Blacklist All Tokens**:
   - Iterate through all refresh tokens for the user (`user_id`).
   - Invalidate each:
     ```bash
     SET refresh_token:<token> "{\"revoked\":true}" EX 0
     ```
   - **Optimization**: Use Redis `SCAN` to avoid full scans:
     ```bash
     SCAN 0 MATCH "refresh_token:*" | FOREACH "SET $KEY \"{\"revoked\":true}\" EX 0"
     ```
2. **Clear Access Tokens**:
   - If using stateless access tokens, they’re already invalidated by TTL.
   - For session-based tokens, invalidate them separately.

---

### **6. Token Blacklisting vs. Rotation**
| **Aspect**               | **Token Blacklisting**                          | **Token Rotation**                          |
|--------------------------|-----------------------------------------------|--------------------------------------------|
| **Mechanism**            | Immediately revoke tokens (e.g., `TTL=0`).     | Replace tokens with new ones (no immediate revocation). |
| **Use Case**             | Critical security events (e.g., password change, brute force). | General session management (e.g., refresh token expiry). |
| **Overhead**             | High (scanning/revoking all tokens).           | Low (only rotate one token).               |
| **Scalability**          | Poor (Redis scans can be slow).               | Scalable (only affects one token).         |
| **Example**              | `/auth/logout` invalidates all tokens.         | `/auth/refresh` replaces refresh token.    |

**When to Use**:
- **Blacklisting**: For irreversible actions (e.g., password reset, account lockout).
- **Rotation**: For routine sessions (e.g., refresh tokens, short-lived access tokens).

---

### **Example Workflow**
1. **User logs in**:
   - Issues `access_token` (TTL=15m) + `refresh_token` (TTL=7d).
2. **User refreshes**:
   - `/auth/refresh` → New `access_token` + new `refresh_token`.
3. **User changes password**:
   - `/auth/logout` → Invalidate all `refresh_tokens` for `user_id`.
4. **User tries to refresh twice**:
   - Redis detects `usage > 1` → Rejects request.

- **Redis**: For token storage/TTL management.

- **Rate Limiting**: To prevent brute-force attacks on `/auth/refresh`.

### 2. Complete Component Implementation
```typescript
import React, { useState, useMemo, useCallback, useEffect } from 'react';
import PropTypes from 'prop-types';

type SortDirection = 'asc' | 'desc' | 'none';

const DataTable = <T extends object>({
  columns,
  data,
  pageSize = 10,
  currentPage = 1,
  onPageChange,
  onSort,
  onRowSelect,
  onSelectAll,
  rowSelection = false,
  selectAll = false,
  controlled = false,
  controlledRows = [],
  controlledPage = 1,
  controlledPageSize = 10,
  controlledSort = { key: undefined, direction: 'none' },
  controlledSelectedRows = new Set<T>(),
}: DataTableProps<T>) => {
  // State for uncontrolled mode
  const [internalPage, setInternalPage] = useState(currentPage);
  const [internalPageSize, setInternalPageSize] = useState(pageSize);
  const [internalSort, setInternalSort] = useState<SortDirection>('none');
  const [internalSelectedRows, setInternalSelectedRows] = useState<Set<T>>(new Set(controlledRows));

  // Calculate pagination
  const totalPages = Math.ceil(data.length / internalPageSize);
  const startIndex = (internalPage - 1) * internalPageSize;
  const paginatedData = data.slice(startIndex, startIndex + internalPageSize);

  // Calculate sorted data
  const sortedData = useMemo(() => {
    const sorted = [...data];
    const { key, direction } = controlledSort || internalSort === 'none' ? internalSort : controlledSort;

    if (key && direction !== 'none') {
      sorted.sort((a, b) => {
        const aValue = a[key as keyof T];
        const bValue = b[key as keyof T];

        if (typeof aValue === 'string' && typeof bValue === 'string') {
          return direction === 'asc' ? aValue.localeCompare(bValue) : bValue.localeCompare(aValue);
        }
        return direction === 'asc' ? (aValue as number) - (bValue as number) : (bValue as number) - (aValue as number);
      });
    }
    return sorted;
  }, [data, internalSort, controlledSort]);

  // Handle row selection
  const handleRowSelect = useCallback((row: T, isSelected: boolean) => {
    if (rowSelection) {
      const newSelectedRows = new Set<T>(internalSelectedRows);
      if (isSelected) newSelectedRows.add(row);
      else newSelectedRows.delete(row);
      setInternalSelectedRows(newSelectedRows);
      if (onRowSelect) onRowSelect(row, isSelected);
    }
  }, [internalSelectedRows, rowSelection]);

  // Handle select all
  const handleSelectAll = useCallback((isSelected: boolean) => {
    if (selectAll) {
      const newSelectedRows = new Set<T>(internalSelectedRows);
      if (isSelected) {
        paginatedData.forEach(row => newSelectedRows.add(row));
      } else {
        newSelectedRows.clear();
      }
      setInternalSelectedRows(newSelectedRows);
      if (onSelectAll) onSelectAll(isSelected);
    }
  }, [internalSelectedRows, selectAll, paginatedData]);

  // Handle page change
  const handlePageChange = useCallback((page: number) => {
    if (!controlled) {
      setInternalPage(page);
      if (onPageChange) onPageChange(page);
    }
  }, [controlled]);

  // Handle sort
  const handleSort = useCallback((key: keyof T, direction: SortDirection) => {
    if (!controlled) {
      setInternalSort(direction);
      if (onSort) onSort(key, direction);

```tsx
import { useState, useEffect, useRef, useCallback } from 'react';

interface SearchResult {
  results: any[]; // Replace with your actual result type
}

const useSearch = (query: string) => {
  const [results, setResults] = useState<SearchResult['results']>([]);
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);
  const [cacheTimeout, setCacheTimeout] = useState<NodeJS.Timeout | null>(null);
  const debounceRef = useRef<NodeJS.Timeout | null>(null);
  const controllerRef = useRef<AbortController | null>(null);

  // Cache
  const cache = new Map<string, SearchResult['results']>();

  // Debounce function
  const debounce = useCallback((fn: () => void) => {
    if (debounceRef.current) {
      clearTimeout(debounceRef.current);
    }
    debounceRef.current = setTimeout(fn, 300);
  }, []);

  // Fetch results
  const fetchResults = useCallback(async () => {
    if (!query.trim()) {
      setResults([]);
      return;
    }

    // Cancel previous request if it exists
    if (controllerRef.current) {
      controllerRef.current.abort();
    }

    controllerRef.current = new AbortController();

    try {
      setIsLoading(true);
      setError(null);

      // Check cache first
      if (cache.has(query)) {
        setResults(cache.get(query)!);
        return;
      }

      const response = await fetch(`/api/search?q=${encodeURIComponent(query)}`, {
        signal: controllerRef.current.signal,
      });

      if (!response.ok) {
        throw new Error('Network response was not ok');
      }

      const data: SearchResult = await response.json();

      // Cache the results
      cache.set(query, data.results);
      setResults(data.results);

      // Set cache timeout
      setCacheTimeout(setTimeout(() => {
        cache.delete(query);
      }, 5 * 60 * 1000));
    } catch (err) {
      if (err.name !== 'AbortError') {
        setError('Failed to fetch results. Please try again.');
      }
    } finally {
      setIsLoading(false);
    }
  }, [query]);

  // Cleanup cache timeout
  useEffect(() => {
    return () => {
      if (cacheTimeout) {
        clearTimeout(cacheTimeout);
      }
    };
  }, [cacheTimeout]);

  // Debounced search
  useEffect(() => {
    debounce(() => {
      fetchResults();
    });
  }, [query, fetchResults]);

  return {
    results,
    isLoading,
    error,
    query,
    // Expose debounce for SearchPage to use
    debounce,
  };
};

export default useSearch;
```

### 4. Why Canceling In-Flight Requests is Important

**Importance of cancelling in-flight requests:**

Cancelling in-flight requests prevents several critical bugs:

1. **Memory Leaks**: Without cancellation, old requests remain in the network queue, consuming memory and potentially causing performance issues.

2. **Duplicate Requests**: If a new search comes in while an old one is still processing, the old request would complete and return stale data, while the new request would start a new fetch. This could lead to inconsistent UI states.

3. **Race Conditions**: Without cancellation, the latest search might overwrite results from previous searches, leading to incorrect UI rendering.

4. **Resource Exhaustion**: If multiple requests are allowed to complete without cancellation, the server could be overwhelmed with duplicate requests for the same query.

5. **Stale Data**: The browser's cache might still contain results from previous searches, and without cancellation, these stale results could be returned before new data is fetched.

**Implementation Details:**
- We use `AbortController` to create a signal that can be used to abort requests
- The controller is created for each search and aborted when a new search starts
- The fetch function checks for the abort signal and throws an `AbortError` if it's cancelled

This ensures that only the most recent search request is processed, providing consistent and up-to-date results to the user.

class CollaborativeDocServer {
  constructor() {
    this.wss = new WebSocket.Server({ port: 8080 });
    this.connections = new Map(); // { docId: { userId: WebSocket } }
    this.userStates = new Map(); // { userId: { cursor: { pos, offset } } }
  }

  async handleConnection(ws) {
    ws.onopen = () => {
      console.log(`User connected: ${ws.upgradeReq.socket.remoteAddress}`);
    };

    ws.onmessage = async (event) => {
      const message = JSON.parse(event.data);

      switch (message.type) {
        case "join":
          this.joinDocument(ws, message);
          break;

        case "text_change":
          this.handleTextChange(ws, message);
          break;

        case "cursor_move":
          this.handleCursorMove(ws, message);
          break;

        case "user_left":
          this.handleUserLeft(ws, message);
          break;
      }
    };

    ws.onclose = () => {
      console.log(`User disconnected: ${ws.upgradeReq.socket.remoteAddress}`);
    };
  }

  joinDocument(ws, { documentId, userId }) {
    if (!connections.has(documentId)) {
      connections.set(documentId, new Map());
    }

    const docConnections = connections.get(documentId);
    docConnections.set(userId, ws);

    // Broadcast join to others
    const others = Array.from(docConnections.values())
      .filter(w => w !== ws)
      .map(w => w.upgradeReq.socket.remoteAddress);

    others.forEach(addr => {
      const targetWs = connections.get(documentId)[addr];
      if (targetWs) targetWs.send(JSON.stringify(message));
    });
  }

  handleTextChange(ws, { documentId, userId, delta, cursor }) {
    // Validate document exists
    if (!connections.has(documentId)) return;

    // Broadcast to others
    broadcastToDocument(ws, { ...message, type: "text_change" });
  }

  handleCursorMove(ws, { documentId, userId, position, offset }) {
    // Update cursor state
    this.userStates.set(userId, { cursor: { position, offset } });

    // Broadcast to others
    broadcastToDocument(ws, { ...message, type: "cursor_move" });
  }

  handleUserLeft(ws, { userId }) {
    // Remove from all documents
    Array.from(connections.values()).forEach(connections => {
      connections.delete(userId);
    });
  }
}

// Initialize server
const server = new CollaborativeDocServer();
server.wss.on('connection', (ws) => server.handleConnection(ws));
server.wss.serve();
```

- **Push Notifications** (e.g., mobile apps) are not ideal for a web-based tool (requires separate infrastructure).

**Implementation Approach:**
- **Primary:** WebSocket for real-time updates (e.g., task assignments, comments).
