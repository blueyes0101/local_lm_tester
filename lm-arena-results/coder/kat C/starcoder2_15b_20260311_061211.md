# Answers
## 1. What transport mechanism would you use and why? (WebSocket, SSE, polling, push notifications  — compare and justify)

I would go with webSocket. 
1. It's fast and responsive and it is the best choice for real-time, bidirectional communication, such as chat apps, multiplayer games, etc. It's the most reliable.
2. It allows for full-duplex communication, it means that the server can send messages to the client without an explicit request.
3. It's a reliable connection that allows for reconnection in case of disconnections.
4. It allows to send data in both directions.
5. It has very low overhead and doesn't require extra HTTP requests to initiate or poll the connection, so it's the fastest way to get real-time updates.
6. It has a binary protocol that's faster than the text-based alternatives.
7. There is no need to worry about Cross-Origin Resource Sharing (CORS) since websockets use the same port and protocol as the page being used.
8. It's a standard technology that's supported by most major browsers.

Source: [1](https://stackoverflow.com/a/11077183/10201022) [2](https://stackoverflow.com/a/22429619/10201022) [3](https://stackoverflow.com/a/19305138/10201022)

## 5. How do you scale this if there are 100,000 concurrent users?

### Vertical scaling

Vertical scaling is when you increase the resources of a single server. The main goal of this approach is to solve a single node's processing or memory capacity.

Vertical scaling is usually limited by the capacity of a single server, which can be a challenge when the demand for resources increases.

### Horizontal scaling

Horizontal scaling is when you increase the number of servers that split the load. Each server is a clone of the original server. Horizontal scaling is the opposite of vertical scaling.

Horizontal scaling is limited by the capacity of each server. To make it effective, your application needs to be able to scale horizontally.

### Load balancer

A load balancer is a tool that optimizes resource utilization, maximize throughput, minimize response time, and avoid overload of a single server. It can be a hardware or a software.

A load balancer can be placed between the clients and the servers. It checks the health of the servers and the client requests are routed to healthy servers.

Source: [1](https://www.nginx.com/blog/nginx-performance-tuning-tips-tricks/) [2](https://www.nginx.com/resources/glossary/load-balancing/)
