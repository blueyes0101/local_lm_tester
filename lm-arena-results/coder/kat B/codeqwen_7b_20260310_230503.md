**A:**
First, we install the necessary dependencies:

```bash

```

Create a `server.ts` file:

```typescript
import express from 'express';
import bodyParser from 'body-parser';
import cookieParser from 'cookie-parser';
import bcrypt from 'bcrypt';
import jwt from 'jsonwebtoken';

const app = express();
app.use(bodyParser.json());
app.use(cookieParser());

// User model
interface User {
  id: string;
  email: string;
  role: string;
}

let users: User[] = [];

// POST /auth/register
app.post('/auth/register', async (req, res) => {
  const { email, password } = req.body;

  // Hash password
  const hashedPassword = await bcrypt.hash(password, 12);

  // Create user
  const user: User = {
    id: Math.random().toString(),
    email,
    role: 'user',
  };
  users.push(user);

  // Return user without password
  res.json({
    id: user.id,
    email: user.email,
    role: user.role,
  });
});

// POST /auth/login
app.post('/auth/login', async (req, res) => {
  const { email, password } = req.body;

  // Find user
  const user = users.find((u) => u.email === email);

  // Check if user exists and password is correct
  if (!user || !(await bcrypt.compare(password, user.password))) {
    res.status(401).json({ error: 'Invalid email or password' });
    return;
  }

  // Create access token
  const accessToken = jwt.sign(
    {
      userId: user.id,
      email: user.email,
      role: user.role,
      iat: Math.floor(Date.now() / 1000),
      exp: Math.floor(Date.now() / 1000) + 15 * 60,
    },
    'secretKey'
  );

  // Create refresh token
  const refreshToken = jwt.sign(
    { userId: user.id, email: user.email, role: user.role },
    'refreshKey',
    { expiresIn: '7d' }
  );

  // Set refresh token as httpOnly cookie
  res.cookie('refreshToken', refreshToken, { httpOnly: true, maxAge: 7 * 24 * 60 * 60 * 1000 });

  // Return access token
  res.json({ accessToken });
});

// GET /auth/me
app.get('/auth/me', authMiddleware, (req, res) => {
  res.json(req.user);
});

// Auth middleware
function authMiddleware(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];

  if (!token) {
    res.status(401).json({ error: 'Missing token' });
    return;
  }

  try {
    const decoded = jwt.verify(token, 'secretKey');

    // Attach user to req.user
    req.user = decoded;

    next();
  } catch (err) {
    res.status(401).json({ error: 'Invalid token' });
  }
}

// Error handling middleware
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).json({ error: 'Internal server error' });
});

app.listen(3000, () => {
  console.log('Server listening on port 3000');
});
```

In this server.ts file, we create a simple authentication system using Express, TypeScript, bcrypt, and jsonwebtoken. The `/auth/register` endpoint creates a new user, the `/auth/login` endpoint generates access and refresh tokens, and the `/auth/me` endpoint checks if the user is authenticated. The `authMiddleware` function is used to validate the access token and attach the user to the request object.

Regarding the refresh token, we store it in an httpOnly cookie instead of localStorage because it's important to keep the refresh token secure and not expose it to the client-side JavaScript code. HttpOnly cookie ensures that the cookie cannot be accessed by client-side scripts, which makes it less susceptible to cross-site scripting (XSS) attacks. This is the main reason for using httpOnly cookie for the refresh token instead of localStorage.

If someone sends an expired token, the JWT library will throw a `TokenExpiredError` error, which we catch in the error handling middleware and return a 401 error with the appropriate message.

import React, { useState, useEffect, useCallback } from 'react';
import { Column, DataTableProps } from './DataTable';

function DataTable<RowType>({
  columns,
  data,
  isControlled = false,
  selectedRows = [],
  onSelectionChange,
  pageSizeOptions = [10, 25, 50],
  pageSize = 10,
  onPageSizeChange,
}: DataTableProps<RowType>) {
  const [sortDirection, setSortDirection] = useState<'asc' | 'desc' | null>(null);
  const [sortedColumn, setSortedColumn] = useState<keyof RowType | null>(null);
  const [currentPage, setCurrentPage] = useState(0);

  const sortedData = useMemo(() => {
    if (!sortDirection || !sortedColumn) return data;
    return [...data].sort((a, B) => {
      if (A[sortedColumn] < B[sortedColumn]) return sortDirection === 'asc' ? -1 : 1;
      if (A[sortedColumn] > B[sortedColumn]) return sortDirection === 'asc' ? 1 : -1;
      return 0;
    });
  }, [data, sortDirection, sortedColumn]);

  const visibleData = useMemo(() => {
    const startIndex = currentPage * pageSize;
    const endIndex = startIndex + pageSize;
    return sortedData.slice(startIndex, endIndex);
  }, [sortedData, currentPage, pageSize]);

  const handleSort = useCallback(
    (columnKey: keyof RowType) => {
      if (sortedColumn === columnKey) {
        setSortDirection(sortDirection === 'asc' ? 'desc' : 'asc');
      } else {
        setSortDirection('asc');
        setSortedColumn(columnKey);
      }
    },
    [sortedColumn, sortDirection]
  );

  const handleSelectAll = useCallback(() => {
    if (selectedRows.length === data.length) {
      if (onSelectionChange) onSelectionChange([]);
    } else {
      if (onSelectionChange) onSelectionChange(data);
    }
  }, [data, selectedRows, onSelectionChange]);

  const handleRowSelection = useCallback(
    (rowData: RowType) => {
      if (isControlled) {
        if (onSelectionChange) onSelectionChange([...selectedRows, rowData]);
      } else {
        setSelectedRows([...selectedRows, rowData]);
      }
    },
    [isControlled, selectedRows, OnSelectionChange]
  );

  useEffect(() => {
    if (onSelectionChange) onSelectionChange(selectedRows);
  }, [selectedRows, onSelectionChange]);

  useEffect(() => {
    if (onPageSizeChange) onPageSizeChange(pageSize);
  }, [pageSize, onPageSizeChange]);

  return (
    <div>
      <table>
        <thead>
          <tr>
            {columns.map((column) => (
              <th
                key={column.key}
                style={{ width: column.width || 'auto' }}
                onClick={() => (column.sortable ? handleSort(column.key) : null)}
              >
                {column.header}
                {column.sortable && (
                  <span>
                    {' '}
                    {sortedColumn === column.key ? (
                      sortDirection === 'asc' ? (
                        <>&#x2191;</>
                      ) : (
                        <>&#x2193;</>
                      )
                    ) : (
                      <>&#x23CE;</>
                    )}
                  </span>
                )}
              </th>
            ))}
          </tr>
        </thead>
        <tbody>
          {visibleData.map((rowData) => (
            <tr
              key={rowData.id}
              onClick={() => handleRowSelection(rowData)}
              style={{ backgroundColor: selectedRows.includes(rowData) ? 'lightgray' : 'white' }}
            >
              {columns.map((column) => (
                <td key={column.key} style={{ width: column.width || 'auto' }}>
                  {column.renderCell ? column.renderCell(rowData) : rowData[column.key]}
                </td>
              ))}
            </tr>
          ))}
        </tbody>
      </table>
      <div>
        <input type="checkbox" checked={selectedRows.length === data.length} onChange={handleSelectAll} />{' '}

      </div>

          <button key={option} onClick={() => onPageSizeChange(option)}>

**A:**
Sure, here's a custom hook and the SearchPage component that use the useSearch hook to handle fetch, debounce, cache, and cancellation:
```javascript
import { useState, useEffect, useRef } from 'react';
import { useDebounce, useLocalStorage } from 'react-use';
import { useHistory } from 'react-router-dom';
const CACHE_KEY = 'search-cache';
const CACHE_EXPIRATION = 5 * 60 * 1000; // 5 minutes in milliseconds
function useSearch(query) {
  const [results, setResults] = useState([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);
  const [cache, setCache] = useLocalStorage(CACHE_KEY, {});
  const controller = useRef(new AbortController());
  const debouncedQuery = useDebounce(query, 300);
  useEffect(() => {
    // Cancel in-flight requests when a new search starts
    controller.current.abort();
    controller.current = new AbortController();
    const signal = controller.current.signal;
    const cachedResults = cache[debouncedQuery];
    if (cachedResults && Date.now() - cachedResults.timestamp < CACHE_EXPIRATION) {
      // Use cached results if available and not expired
      setResults(cachedResults.results);
      setLoading(false);
      setError(null);
    } else {
      setLoading(true);
      setError(null);
      setResults([]);
      fetch(`/api/search?q=${debouncedQuery}`, { signal })
        .then(response => response.json())
        .then(data => {
          // Save results to cache
          const newCache = { ...cache };
          newCache[debouncedQuery] = {
            timestamp: Date.now(),
            results: data
          };
          setCache(newCache);
          setResults(data);
          setLoading(false);
          setError(null);
        })
        .catch(error => {
          if (error.name === 'AbortError') {
            // Ignore error if request was cancelled
            return;
          }
          setLoading(false);
          setError(error);
        });
    }
  }, [debouncedQuery, cache, setCache]);
  return { results, loading, error };
}
function SearchPage() {
  const [query, setQuery] = useState('');
  const { results, loading, error } = useSearch(query);
  const history = useHistory();
  useEffect(() => {
    history.push({
      pathname: '/search',
      search: `?q=${query}`
    });
  }, [query, history]);
  return (
    <div>
      <input
        type="text"
        value={query}
        onChange={event => setQuery(event.target.value)}
      />
      {loading && <div>Loading...</div>}
      {error && <div>Error: {error.message}</div>}
      {!loading && !error && results.length === 0 && <div>No results</div>}
      {!loading && !error && results.length > 0 && (
        <ul>
          {results.map(result => (
            <li key={result.id}>{result.name}</li>
          ))}
        </ul>
      )}
    </div>
  );
}
export default SearchPage;
```
Here's why cancelling in-flight requests is important here: Without cancelling in-flight requests, the following could happen:
- If the user types a new search before the previous one is done, the previous request could be cancelled and the new request would start.
- If the user clicks the back button before the previous request is done, the previous request could still be in flight and the user would see an empty state until the request is done.
By cancelling in-flight requests, we prevent the above issues and ensure a better user experience.

## Question 9
**Q:** You are building a collaborative document editor (like Google Docs, simplified):
- Multiple users can edit the same document simultaneously
- Each user sees others' cursors in realtime
- Changes are broadcast to all connected clients
- If a user disconnects and reconnects, they get the latest document state

Implement the server side in Node.js using the ws library:
1. Connection handling — how do you associate a WebSocket connection with a user and document?
2. Message protocol — define the message types (JSON) for: join document, text change, cursor move, user left
3. Broadcasting — when user A makes a change, how do you send it to all other users in the same document but NOT back to user A?
4. Reconnection — how do you handle a user rejoining after disconnect?
5. What happens if two users edit the exact same position simultaneously? You don't need to implement OT/CRDT — just explain the problem and name the solution approaches.

Write the core server implementation (connection map, message routing, broadcast logic).

**A:**
Here is a simple implementation of the server-side code using the ws library in Node.js:

```javascript
const WebSocket = require('ws');

const wss = new WebSocket.Server({ port: 8080 });

const connections = new Map();
const documents = new Map();

wss.on('connection', (ws, request) => {
  const userId = request.headers['user-id'];
  const documentId = request.headers['document-id'];

  if (!userId || !documentId) {
    ws.close(1008, 'Missing user-id or document-id header');
    return;
  }

  if (!connections.has(userId)) {
    connections.set(userId, new Set());
  }

  const userConnections = connections.get(userId);
  userConnections.add(ws);

  if (!documents.has(documentId)) {
    documents.set(documentId, new Set());
  }

  const documentConnections = documents.get(documentId);
  documentConnections.add(ws);

  ws.on('message', (message) => {
    const data = JSON.parse(message);
    switch (data.type) {
      case 'join':
        sendMessage(ws, { type: 'join', userId });
        sendMessage(ws, { type: 'document', documentId });
        break;
      case 'text':
        sendMessage(ws, { type: 'text', userId, text: data.text });
        break;
      case 'cursor':
        sendMessage(ws, { type: 'cursor', userId, position: data.position });
        break;
      case 'leave':
        userConnections.delete(ws);
        documentConnections.delete(ws);
        ws.close();
        break;
      default:
        ws.close(1008, 'Invalid message type');
    }
  });

  ws.on('close', () => {
    userConnections.delete(ws);
    documentConnections.delete(ws);
  });
});

function sendMessage(ws, message) {
  ws.send(JSON.stringify(message));
}
```

1. Connection handling — associating a WebSocket connection with a user and document is done by using the headers of the WebSocket connection. The `user-id` and `document-id` headers are used to identify the user and document respectively.

2. Message protocol — the server defines the message types (JSON) for `join`, `text`, `cursor`, and `leave`. When a user joins a document, the server sends a `join` message to the user with their `user-id` and sends the current `document-id` to the user. When a user makes a change to the document, the server sends a `text` message to all other users in the document. When a user moves their cursor in the document, the server sends a `cursor` message to all other users in the document. When a user leaves the document, the server sends a `leave` message to all other users in the document.

3. Broadcasting — when a user makes a change to the document, the server sends a `text` message to all other users in the document but NOT back to the user who made the change. This is done by looping through all the users in the document and checking if the user is not the one who made the change.

4. Reconnection — when a user rejoins after disconnect, the server simply adds the user back to the document and sends them the current document state.

5. Simultaneous edits — if two users edit the exact same position simultaneously, the last edit will overwrite the other edit because the server only broadcasts the latest document state to all other users in the document.

The solution approaches for handling simultaneous edits are:
- "Concurrency control" - this approach involves using a locking mechanism to ensure that only one user can make a change at a time. This can be achieved using a database that supports transactions.
- "Conflict resolution" - this approach involves resolving conflicts between different changes. this can be achieved using a conflict resolution algorithm that prioritizes the changes made by the user who made the change last.
